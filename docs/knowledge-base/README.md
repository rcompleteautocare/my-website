# R Complete Auto Care — Shop Knowledge Base

The single source of truth for shop operations, standards, and procedures.
Anything an employee, manager, or AI assistant needs to reference when writing
training materials, onboarding documents, checklists, or new SOPs lives here.

## How to use this

- **Employees / managers:** Read the relevant SOP. Managers own the daily
  checklists.
- **AI assistants:** When generating employee training materials, onboarding
  documents, or checklists, treat the documents in this folder as authoritative
  source content. Follow the formatting and standards of existing SOPs when
  producing new ones. Do not invent policy — if something isn't covered here,
  flag it rather than guessing.

## Conventions for new documents

1. **Location:** SOPs go in [`sops/`](sops/). Reference material (scripts,
   templates, checklists) can get its own subfolder as the base grows.
2. **Front matter:** Start every document with title, version, effective date,
   and owner (see the SOP for the pattern).
3. **Version control:** Bump the version on any substantive change and add a
   line to the document's Revision History. Update the version/date in the table
   above.
4. **Formatting:** Markdown, mobile-readable, one `#` H1 per document, clear
   section headings, scannable bullets and checkboxes. Match the tone and
   structure of the existing SOP.
5. **Index:** Add every new document to the Contents table above.
