# R Complete Auto Care — Website Codebase Guide

## Project Overview

Marketing website for **R Complete Auto Care**, a diagnostic-first auto repair shop in Crown Point, Indiana. Built with Next.js 16 App Router, React 19, Tailwind CSS v4, and TypeScript. The site is content-heavy, SEO-optimized, and conversion-focused (driving phone calls and service requests).

## Architecture & Conventions

### Routing

- Uses **Next.js App Router** (app/ directory).
- All routes are static content pages — no dynamic `[slug]` routes.
- No API routes or route handlers.
- No middleware.

### Page Types

1. **Service pages** (`/services/*`) — Deep dives into specific repair services with symptoms, process, FAQ, and related links.
2. **Location pages** (`/auto-repair-{city}-in/`) — City-specific landing pages for SEO, cross-linked to each other.
3. **Symptom pages** (`/car-wont-start-*`, `/car-overheating-*`, etc.) — Emergency/problem-focused pages targeting search queries.
4. **Core pages** — Home, About, Contact.

### Metadata & SEO

Every page exports a `metadata` object using the Next.js Metadata API:

```typescript
export const metadata = {
  title: "Page Title | R Complete Auto Care",
  description: "...",
  alternates: { canonical: "https://www.rcompleteautocare.com/..." }
};
```

- The root `layout.tsx` contains JSON-LD structured data (schema.org `AutoRepair` + `LocalBusiness`).
- Some individual pages also embed their own JSON-LD `<script>` blocks.
- `app/sitemap.ts` defines all routes for sitemap generation.
- **When adding a new page, always add its URL to `app/sitemap.ts`.**

### Styling

The codebase uses a **mixed styling approach**:

- **`globals.css`** imports Tailwind v4 via `@import "tailwindcss"` and defines custom CSS classes (`.cta-button`, `.feature-grid`, `.faq-list`, etc.) plus responsive media queries.
- **Most pages use inline styles** (`style={{ }}`) rather than Tailwind utility classes or CSS modules.
- **Symptom pages** (`.jsx` files) use CSS classes from `globals.css` instead of inline styles.
- **Nav.tsx** embeds a `<style>` tag with component-scoped CSS and media queries.
- No CSS modules are used anywhere.
- **Color palette:** Dark backgrounds (#1a1a1a, #111), Red accents (#e63946, #c0392b), Greens for trust (#27ae60), Gray shades.

### Components

Only three shared components exist in `app/components/`:

| Component          | Type            | Notes |
|--------------------|-----------------|-------|
| `Nav.tsx`          | Client ("use client") | Header, mobile hamburger menu, services dropdown, trust bar |
| `Footer.tsx`       | Server          | 4-column footer with business info, service links, location links, hours |
| `FaqAccordion.jsx` | Client ("use client") | Accepts `faqs` array of `{q, a}` objects |

Pages are self-contained — most content is directly in page files with inline styles rather than extracted into components.

### File Type Conventions

- **TypeScript (`.tsx`)** is used for service pages, location pages, core pages, Nav, and Footer.
- **JavaScript (`.jsx`)** is used for symptom/emergency pages and FaqAccordion.
- New pages should use **TypeScript (`.tsx`)**.

### External Integrations

| Service   | Purpose          | Integration Point |
|-----------|------------------|-------------------|
| Formspree | Contact form     | POST to `https://formspree.io/f/xykbqraj` in `contact/page.tsx` |
| Elfsight  | Google Reviews   | Script loaded in `layout.tsx`, widget div in `page.tsx` |
| Google Maps | Map embed      | iframe in `contact/page.tsx` |

No environment variables are required — all integrations use hardcoded public keys/URLs.

## Configuration Details

### next.config.ts

- Redirects `rcompleteautocare.com` (non-www) to `www.rcompleteautocare.com` (permanent).
- Redirects `/schedule-service` to `/contact` (permanent).

### tsconfig.json

- Strict mode enabled.
- Path alias: `@/*` maps to project root.
- Target: ES2017, module resolution: bundler.

### ESLint

- Flat config format (`eslint.config.mjs`).
- Extends `eslint-config-next/core-web-vitals` and `eslint-config-next/typescript`.
- Ignores: `.next/`, `out/`, `build/`, `next-env.d.ts`.

### Sitemap

- `next-sitemap.config.js` sets `siteUrl` to `https://www.rcompleteautocare.com`.
- Generates `robots.txt` and XML sitemaps on `npm run postbuild`.
- `app/sitemap.ts` also provides a programmatic sitemap (both coexist).

## Development Guidelines

1. **Read Next.js docs first.** Per AGENTS.md, check `node_modules/next/dist/docs/` before writing Next.js code — APIs may differ from training data.
2. **No test suite exists.** Verify changes by running `npm run build` and checking the dev server (`npm run dev`).
3. **Run `npm run lint`** before committing to catch ESLint errors.
4. **Match existing page patterns.** When creating new service or location pages, mirror the structure and metadata pattern of existing ones.
5. **Keep pages self-contained.** The codebase convention is large page files with inline content rather than extracted sub-components.
6. **Update sitemap.** When adding new routes, add them to `app/sitemap.ts`.
7. **Preserve SEO elements.** Every page must have a `metadata` export with `title`, `description`, and ideally `alternates.canonical`.
8. **Use TypeScript** for new files (`.tsx` over `.jsx`).
9. **No heavy dependencies.** This is a lightweight marketing site — avoid adding large libraries.
10. **Maintain business accuracy.** Hours, phone number, address, and service descriptions must stay consistent with the business info above.
